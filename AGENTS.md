# TrafficLab Agent Guide

- TrafficLab turns concurrency problems into measurable experiments.
- Do not build a generic CRUD app.
- Do not add PR review or GitHub App features.
- Do not fake metrics.
- Every metric must come from actual run data.
- Keep MVP focused on concert booking first.
- Prefer simple, working vertical slices over broad unfinished features.
- Backend correctness is more important than visual polish.
- Any concurrency strategy must include a short explanation and test coverage.
- README must stay updated when architecture or commands change.
